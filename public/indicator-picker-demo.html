Picking from many indicators — in the real ConditionRow context

Current reality: indicators are picked from a native <select> inside each ConditionRow — one row per rule. The chip tray above only shows what's already been referenced. Today the dropdown lists 5 candle channels + ~6 indicators = 11 options; once the catalog hits ~150 (talib + pandas_ta) that native select breaks. Below are 3 directions, each drawn in the actual drawer surroundings with the real tokens (#14181f / #181c22 / #F0B90B).

V1

Searchable combobox replaces the native <select>

Smallest delta: same row, same flow — only the left/right metric dropdown becomes a searchable popover with category groups. Single pick per row. Multi-add isn't here; chip tray still grows implicitly from the rows.

V2

Multi-select chip-tray + auto-seed rows

Adds an explicit Add indicators button on the chip tray. Opens a multi-checkbox popover. On apply, each picked indicator (a) appears as a chip and (b) seeds an empty ConditionRow under "IF" so the user just fills operator/value.

V3

Typeahead chip + short row select

Decouples "what's in the strategy" from "what this row references." Add metrics via typeahead chip (no modal). The row's metric select becomes short — only the chips you've already added — so 150 items never explode into one row.

Recommendation

  • Ship V1 first — it's the smallest change that unblocks the 150-item catalog. Same Popover primitive you already use for IndicatorChip, plugs into both left + right ConditionRow selects.
  • Stack V2 on top later, only after we see users add ≥3 indicators per strategy (Cypheus / analytics signal). The "seed rows" action is the real value-add.
  • V3 is the most opinionated rewrite — defer unless we want the chip tray to be the canonical add path.
